================================================================================
                    STEP 1: SIGN UP ON RENDER (2 minutes)
================================================================================

1. Open your browser and go to:
   https://render.com

2. Click "Sign Up" button (top right)

3. Select "Sign up with GitHub"
   (This is the easiest method)

4. You'll see a GitHub authorization popup
   Click "Authorize Render"

5. You're now logged into Render!
   You'll see the dashboard with a blue "New +" button

DONE! ✅ Move to Step 2

================================================================================
               STEP 2: CREATE POSTGRESQL DATABASE (5 minutes)
================================================================================

IMPORTANT: Create database FIRST, before web service!

1. In Render dashboard, click "New +" button (top right, blue)

2. From dropdown menu, select "PostgreSQL"
   (NOT "Web Service" - select "PostgreSQL")

3. You'll see a form. Fill in these exact values:

   Name: fileflow_db
   
   Database: fileflow
   
   User: fileflow_user
   
   Region: oregon
   
   PostgreSQL Version: (keep default)
   
   IMPORTANT: At the bottom, make sure "Free" plan is selected
   (Don't select "Starter" or paid options)

4. Click "Create Database" button

5. Wait 1-2 minutes while database is provisioned
   You'll see status: "Creating..." → "Available"

6. CRITICAL STEP: Copy the connection string
   - Click on your database (fileflow_db)
   - Look for "Connection" section
   - You'll see "Internal Database URL" and "External Database URL"
   - Copy the "External Database URL"
   - Example: postgresql://user:password@host:port/database
   - SAVE THIS! You'll need it in Step 3

DONE! ✅ Move to Step 3

================================================================================
                  STEP 3: CREATE WEB SERVICE (5 minutes)
================================================================================

1. Go back to Render dashboard
   Click "New +" button again

2. From dropdown menu, select "Web Service"
   (This time it's Web Service, not PostgreSQL)

3. Connect your GitHub repository:
   
   a. Enter your repository URL:
      https://github.com/YOUR_USERNAME/FileFlow.git
   
   Replace YOUR_USERNAME with your actual GitHub username!
   
   Example: https://github.com/amitdevx/FileFlow.git

4. Click "Connect" button

5. GitHub will ask for permission
   Click "Authorize Render"

6. Select the FileFlow repository (if prompted to choose)

7. Now you'll see the configuration form:

   === FILL IN THESE FIELDS ===
   
   Name: fileflow
   
   Root Directory: (leave blank)
   
   Branch: main
   (Make sure "main" is selected, not "master")
   
   Environment: Python
   (Auto-selected)
   
   Build Command (EXACT - copy and paste):
   pip install -r FileFlow/backend/requirements.txt && cd FileFlow && python -c "from backend.app import app, db; app.app_context().push(); db.create_all()"
   
   Start Command (EXACT - copy and paste):
   cd FileFlow && gunicorn -w 4 -b 0.0.0.0:$PORT "backend.app:app"
   
   === SELECT PLAN ===
   
   At the bottom, select "Free" plan
   (NOT "Standard" or paid plans)

8. Click "Advanced" (expand section at bottom)

9. Add Environment Variables:
   
   First variable:
   Key: FLASK_ENV
   Value: production
   Click "Add Environment Variable"
   
   Second variable:
   Key: DATABASE_URL
   Value: (PASTE the connection string from Step 2)
   Example: postgresql://user:password@host:5432/fileflow
   
   Leave other variables blank

10. Click "Create Web Service" button

    Deployment will start automatically!

DONE! ✅ Move to Step 4

================================================================================
                      TROUBLESHOOTING GUIDE
================================================================================

❌ PROBLEM: Deployment fails with error messages

SOLUTION:
1. Click on service
2. Click "Logs" tab
3. Read the error message
4. Common errors:
   - "ModuleNotFoundError" → Check requirements.txt
   - "ImportError" → Check start command path
   - "Connection refused" → Wait for PostgreSQL (1-2 min)

❌ PROBLEM: Can't access app URL

SOLUTION:
1. Wait 30 seconds
2. Hard refresh browser: Ctrl+Shift+Delete (Windows) or Cmd+Shift+Delete (Mac)
3. Clear browser cache completely
4. Try in incognito/private window
5. Check status in Render shows "Live" ✅

❌ PROBLEM: Database won't connect

SOLUTION:
1. Wait 1-2 minutes for PostgreSQL to fully start
2. Check DATABASE_URL environment variable is correct
3. Verify connection string was copied correctly
4. Render will auto-retry connections

❌ PROBLEM: "502 Bad Gateway" error

SOLUTION:
1. App is still starting (normal for first deploy)
2. Wait 2 minutes
3. Refresh page
4. Try again

❌ PROBLEM: No CSS/styling (page looks plain/broken)

SOLUTION:
1. Hard refresh: Ctrl+Shift+Delete
2. Clear all browser cache
3. Try different browser
4. Try incognito mode

❌ PROBLEM: Files I uploaded disappeared

SOLUTION:
1. This is NORMAL on free tier
2. Free tier has "ephemeral storage" (temporary)
3. Files reset weekly automatically
4. To keep files permanently:
   - Upgrade to paid tier ($7/month)
   - OR use AWS S3 for storage

❌ PROBLEM: App is very slow / times out

SOLUTION:
1. This is NORMAL on free tier
2. First request can take 30+ seconds (cold start)
3. Subsequent requests are faster
4. To fix:
   - Upgrade to paid tier ($7/month) for "always-on"
   - Or just wait for initial load
